## Description

Decide whether storage-local .maat cache files are only excluded by Maat sync pathspecs or also ignored by the storage repo itself, then make tooling and docs match that decision.

## Acceptance

- The project explicitly decides whether setup should create or maintain a storage .gitignore entry for .maat/.
- If .gitignore enforcement is chosen, setup or setup doctor creates or validates the ignore entry without touching product repos.
- If pathspec-only exclusion is chosen, docs and tests state that normal Git commands can still see .maat as local untracked or modified cache files.
