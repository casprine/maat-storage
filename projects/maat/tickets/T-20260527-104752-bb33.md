## Description

Refine the primary TUI path so a human can see all projects, open one into a Kanban-style board, move through board items, and read a focused detail view with title, description, acceptance criteria, status, ownership, related goal, and recent events.

## Acceptance

- Project list remains the first TUI screen for project-management use.
- Opening a project leads to a board-like ticket view, and opening a ticket leads to a readable detail view.
- The ticket detail includes description, acceptance criteria, current status, owner or claim information, related goal, and recent comments or completion evidence when available.
- Back navigation returns from detail to board and from board to project list.
