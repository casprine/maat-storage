## Description

Align the documented processing states with API, SDK, worker, and UI behavior so agents and users can rely on one status model and useful processing history.

## Acceptance

- Document status values are intentionally chosen and reflected consistently in OpenAPI, SDK types, API responses, worker updates, and UI labels.
- Job complete/fail updates preserve enough step/event history for document detail and troubleshooting.
- Any migration or compatibility concern is documented before changing persisted values.
- Contract tests cover the chosen status values and event visibility.
