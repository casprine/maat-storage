## Description

Preserve the backend-owned contract flow from apps/api/openapi.json through Orval-generated packages/sdk to apps/web, and keep route parameter and response conventions stable for agents and frontend code.

## Acceptance

- Backend route or response changes update OpenAPI in apps/api.
- Generated SDK in packages/sdk is regenerated when OpenAPI changes affect frontend consumers.
- OpenAPI smoke verifies required public, protected, admin, upload, ingestion, and job routes.
- Route params remain kebab-case and downloads/previews stay behind API authorization.
